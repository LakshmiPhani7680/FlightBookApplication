Acceptance criteria <br>

1. RESTful Architecture <br>
   ---Develop a RESTful API using Node.js, ExpressJS, and MongoDB for efficient data handling and routing control.<br>
2. Code Modularity <br>
   ---Organize code using ES6 Modules for maintainability and modularity.
   ---Secure routes so only authenticated users can book, view, or cancel.
3. <b>User Authentication</b>
   <ul>
      <li>Implement a user authentication system with essential features like register, login, and logout.</li>
      <li>Registration should include user details such as name, email, password, role, profilePicture, createdAt and gender.</li></ul></pre>
4. <pre style="font-weight:normal;"><b>User Management</b>
   <br>
   <ul>
   <li>Implement CRUD operations for flights.</li>
   <li>Ensure that each post references the user who created it.</li>
   <li>Post can be updated or deleted only by the post owner.</li></ul></pre>
5. <pre style="font-weight:normal;"><b>flight Management</b>
   <ul>
   <li>Develop a flight booking system that allows users to create, update, fetch and cancel bookings.</li>
   <li>Search flights by origin, destination, date range, price range, and airline.</li>
   <li>Support image upload for each flight (e.g., airline logo) via Multer with file-type validation.</li>
6. <pre style="font-weight:normal;"><b>Booking Functionality</b>
   <ul>
   <li>Create bookings that allow multiple passengers per booking.</li>
   <li>Track Booking Details for user</li><br>
   <li>Automatically assign and decrement available seats.</li>
   <li>Allow cancellations with updated seat counts &amp; status.</li></ul></pre>
7. <pre style="font-weight:normal;"><b>Booking Features</b>
   <ul>
   <li>Booking cancellation &amp; status will be updated by the admin only.</li>
   <li>Booking cancellation request can be raised by the user itself.</li></ul></pre>
8. <pre style="font-weight:normal;"><b>User Profile Updates</b>
   <ul>
   <li>Enable users to update their profiles, including fields like name, email, password or profilePicture.</li>
   <li>Users can view and update their profile, including uploading a profile picture.</li>
   <li>Implement image uploads for user profiles.</li></ul></pre>
